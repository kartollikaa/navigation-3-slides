### Сверенные факты по API (важно для точности доклада)
- Бэкстек — обычный список, которым владеете вы: `add(key)` = вперёд, `removeLastOrNull()` = назад.
- Декоратор VM называется **`rememberViewModelStoreNavEntryDecorator`** (старые альфа-имена `rememberSceneSetupNavEntryDecorator` / `rememberSavedStateNavEntryDecorator` устарели; на стабильной версии — `rememberSaveableStateHolderNavEntryDecorator` + `rememberViewModelStoreNavEntryDecorator`).
- `NavDisplay(sceneStrategies = listOf(...))` — это **список**; первая стратегия, вернувшая `Scene`, побеждает; иначе `SinglePaneSceneStrategy`.
- `BottomSheetSceneStrategy` — **копируемый рецепт** (android/nav3-recipes), в core-библиотеке его нет. `DialogSceneStrategy` — встроена.
- `ListDetailSceneStrategy` — из `androidx.compose.material3.adaptive:adaptive-navigation3`.
- Официальная позиция: по умолчанию **один root NavDisplay**; несколько back-стеков (табы) — это всё ещё один NavDisplay.

## 3. Нарратив миграции (ключевое решение)

Мигрируем **от листьев к корню**, не начиная с хоста:
1. Сначала — **отдельная фича «Каталог»** с внутренней навигацией и bottom sheet; её sheet переезжает в фичу (локальный NavDisplay + BottomSheetSceneStrategy).
2. Затем — **auth** (вложенный флоу phone → sms → name) на один NavDisplay.
3. **Диалог (confirm) остаётся в root** до самого конца.
4. **Последним** мигрируем навигацию **home/host** на единый root NavDisplay; диалог становится `DialogSceneStrategy` в root.

Каждый шаг миграции на слайдах сопровождается блоком кода **ДО → ПОСЛЕ** в две колонки.
